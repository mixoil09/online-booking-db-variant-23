# Онлайн-запись в велоремонт

**Студент:** Утин М.А., группа 454  
**СУБД:** MySQL 8.0  
**Описание:** Система для записи клиентов на ремонт велосипедов с учётом простых/сложных услуг, опыта мастеров, аналогов запчастей.

## 📌 Описание проекта

Проект представляет собой **веб-приложение для онлайн-записи в велоремонтную мастерскую**. Разработана реляционная база данных, уровень доступа к данным (DAL) на PHP с использованием PDO и полноценный веб-интерфейс для управления справочниками.

### Бизнес-правила (вариант 23)

- Услуги делятся на **простые** и **сложные**.
- Сложный ремонт может выполнять только мастер с **опытом более 2 лет**.
- Каждый мастер имеет **приоритет заказов** (от 1 до 10, где 1 – высший).
- Велосипед клиента характеризуется: тип рамы, размер колёс, тип тормозов.
- Запасные части имеют **аналоги** (связь с той же таблицей).
- Нельзя удалить клиента, мастера или услугу, если они используются в записях (внешние ключи `RESTRICT`).

### Основные таблицы

- `Client` – клиенты
- `Bike` – велосипеды
- `Service` – услуги (simple/complex)
- `Mechanic` – мастера (опыт, приоритет)
- `Appointment` – записи на ремонт
- `Part` – запасные части
- `PartAnalog` – аналоги (самоссылка)
- `OrderPart` – использованные детали

### Ключевые ограничения

- `UNIQUE (mechanic_id, app_datetime)` – запрет двойной записи.
- `CHECK (experience_years > 2)` для сложного ремонта (в триггере / приложении).
- Внешние ключи `ON DELETE RESTRICT` – защита от каскадного удаления важных данных.

### SQL-скрипт

Файл `sql/schema.sql` содержит полный скрипт создания БД, таблиц, индексов и тестовых данных.

## 🔧 Уровень доступа к данным (DAL, вторая часть)

Реализован на PHP с использованием **PDO** и паттерна **Repository**.

### Ключевые классы

- `Database` (Singleton) – управление подключением.
- `AbstractRepository` – базовые CRUD-методы (`findAll`, `findById`, `delete`).
- `ClientRepository`, `ServiceRepository`, `MechanicRepository` – специфические запросы.

### Безопасность

- Все запросы – **подготовленные выражения** (защита от SQL-инъекций).
- Сортировка через **белый список** полей.
- Пароли в `config.php` (файл в `.gitignore`).
- **CSRF-токены** в формах.

### Транзакции

В методе `createAppointmentWithParts` (опционально) показано использование транзакций для атомарного создания записи и списания деталей.

### Демонстрационный скрипт

`demo.php` демонстрирует все методы репозиториев. Запуск через браузер или консоль.

## 🌐 Пользовательский интерфейс (третья часть)

Веб-интерфейс для управления **трёмя справочниками**:

- **Клиенты**
- **Услуги**
- **Мастера**

### Реализованный функционал

- 📋 **Список** с пагинацией (10 записей на страницу)
- 🔍 **Поиск** по текстовым полям
- ⬆️⬇️ **Сортировка** по любому столбцу (кликабельные заголовки)
- ➕ **Создание** записи (с валидацией на сервере)
- ✏️ **Редактирование**
- ❌ **Удаление** с проверкой внешних ключей (запрет удаления, если есть связанные записи)
- 👁️ **Детальный просмотр** `view.php` (для совместной работы)
- 💬 **Flash-сообщения** об успехе/ошибке
- 🛡️ **CSRF-защита** всех форм
- 📱 **Адаптивная вёрстка** (минимальная, но чистая)

### Архитектура

Простая реализация **MVC** без фреймворков:
- Маршрутизация через `index.php?entity=client&action=list`
- Контроллеры наследуют `BaseController` (flash, redirect, CSRF).
- Представления используют минимальный PHP и `htmlspecialchars()`.
