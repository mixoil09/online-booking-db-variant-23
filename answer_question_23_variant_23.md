**Формулировка вопроса:**  
Опишите разницу между представлением (VIEW) и материализованным представлением. Могли бы вы использовать VIEW для упрощения вашего первого запроса (с тремя JOIN)?

## Развёрнутый ответ

**Представление (VIEW)** в MySQL — это сохранённый запрос, который выглядит как виртуальная таблица. Оно не хранит данные физически, а при каждом обращении выполняет подлежащий SELECT и возвращает актуальные данные из базовых таблиц. Представления удобны для упрощения сложных запросов, ограничения доступа к столбцам и создания абстракций.

**Материализованное представление (Materialized View)** физически хранит результат запроса на диске. В стандартном MySQL такая возможность отсутствует (есть только в Oracle, PostgreSQL, SQL Server). Однако можно эмулировать материализованное представление через обычную таблицу и периодическое её обновление (например, через событие или триггер). Материализованные представления дают выигрыш в скорости чтения, но требуют дополнительного места и синхронизации при изменениях данных.

**Ключевые различия:**

| Характеристика | VIEW | Материализованное представление (в СУБД, где есть) |
|----------------|------|---------------------------------------------------|
| Хранение данных | нет (виртуально) | да (физически) |
| Скорость чтения | при каждом запросе выполняется исходный SELECT | мгновенно, так как данные уже есть |
| Актуальность | всегда свежие данные | требует обновления (REFRESH) |
| Возможность индексации | нельзя создать индекс | можно создавать индексы |
| Поддержка в MySQL | да (с 5.x) | нет (только эмуляция) |

**Применение VIEW для упрощения первого запроса**

В моей схеме (вариант 23, велоремонт) первый запрос выводит завершённые заказы с данными клиентов, велосипедов, услуг и мастеров. Он содержит четыре JOIN. Создав представление `CompletedRepairsView`, я могу значительно упростить последующие запросы, не повторяя длинные связки.

```sql
CREATE VIEW CompletedRepairsView AS
SELECT 
    a.app_id,
    c.full_name AS client,
    b.frame_type,
    b.wheel_size,
    b.brake_type,
    s.service_name,
    s.type AS service_type,
    m.full_name AS mechanic,
    m.experience_years,
    a.app_datetime,
    a.actual_minutes
FROM Appointment a
JOIN Client c ON a.client_id = c.client_id
JOIN Bike b ON a.bike_id = b.bike_id
JOIN Service s ON a.service_id = s.service_id
JOIN Mechanic m ON a.mechanic_id = m.mechanic_id
WHERE a.status = 'завершено';
